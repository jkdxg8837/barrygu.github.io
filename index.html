OMP_Bug

Posted on 2023-09-27 In Bug fix

Just now, when I was implementating X-VLM, I met the bug reported as following:

OMP: Error #15: Initializing libiomp5.dylib, but found libomp.dylib already initialized.
OMP: Hint This means that multiple copies of the OpenMP runtime have been linked into the program. That is dangerous, since it can degrade performance or cause incorrect results. The best thing to do is to ensure that only a single OpenMP runtime is linked into the process, e.g. by avoiding static linking of the OpenMP runtime in any library. As an unsafe, unsupported, undocumented workaround you can set the environment variable KMP_DUPLICATE_LIB_OK=TRUE to allow the program to continue to execute, but that may cause crashes or silently produce incorrect results. For more information, please see http://www.intel.com/software/products/support/.

However, I haven’t change any bit of my environment. After uninstalling and installing several package and add another config at the heading were also not helping. Finally, changing the order of ‘import numpy ‘ and ‘import torch’ helps me out. So the following coding can fix this bug:

import torch
import numpy as np

The second project in my undergraduate time

Posted on 2023-09-26 In Conversation Street

My second project started with my curiosity towards multi-modal learning. The reason is that I had a bold thought about building a human-like bot that can react like a human, and the ability to understand multimodal information is indispensable. At first, in order to catch up with the senior Ph.D. student, I read papers in multimodal learning and found that these papers are easy to read, to understand, and easy to implement(due to the plug-in-and-play fashion), but hard to organize a new thought of this task. The reason why I have such a difficult time is, I guess, maybe I’m so persuaded by these papers that I can’t see the problem which is not solved yet.

Strangely though, after just plugging in several modules into my old-fashioned baseline, it works sanguinely well and even surpassed the performance of a fancy Pre-trained model. As I’m not considering this work in a problem-driven style, I had a really tough time writing the first several sections of this paper. After carefully fabricating the problem and motivation, I noticed that I have a piercing eye, which I can see through the authors’ rhetoric of their problems and motivations. This made it easier to categorize papers, for I could have a global overlook of them, instead of just being lured by their fancy writing style and graph of model structure.

The first project in my undergraduate time

Posted on 2023-09-21 Edited on 2023-09-26 In Conversation Street

As I don’t want to tell the same story(research proposal or research experiences) shown in my CV or in E-mail, I would like to share with you the progress of my own research career. If you are not interested in me after reading through my CV, just ignore this part.

The first project I received in my master’s career could be hardly defined as a research-oriented project. It’s a project that we need to build a dazzling little drone that would able to auto-detect enemies in an enclosed indoor environment and destroy them by a rail gun. To find a proper algorithm that can explore the environment by itself while avoiding some hypothetical obstacles, I started to read papers in SLAM and other drone algorithms, only focusing on whether the algorithm can fit our needs and whether it can be deployed on our poor platform. I don’t know any innovative points of these papers, only to find out a proper SOTA method that is easy to implement. Luckily, I have had some success in my simulation platform, but it frustratingly failed in the real world because of the terrible quality of the drone’s eye(Binocular camera). After a huge mess, our whole team attended a competition and after winning a little award, we transmitted this project to the juniors with no regret.

Also, to prove my words, here is the repo of the GPS waypoint algorithm. It can run in ROS and can navigate to a nearby GPS location automatically only with the input datastream of a GPS receiver. That is to say, it doesn’t need to synchronize with the pose of the drone, which makes it a simple but effective algorithm.
